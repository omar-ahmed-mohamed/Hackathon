Aspect-Based Sentiment Analysis (ABSA)

Build a production-style Arabic ABSA system that can understand noisy, real customer feedback and score each mentioned aspect with the correct sentiment.
Challenge Objective

This task goes beyond traditional sentiment classification. For each review, your model is expected to identify multiple aspects within the same text and evaluate each one independently as positive, negative, or neutral. This approach allows for a more nuanced and insightful understanding of user feedback, enabling you to build a more sophisticated and impactful model.
1

Identify all aspects discussed in each review (multi-label). One review may mention food, service, and price together.
2

Classify sentiment per aspect (positive, negative, neutral), even when sentiments are mixed in one sentence.
3

Generate valid JSON predictions for the hidden test file exactly in the required submission schema.
4

Design for generalization across many sectors (restaurants, delivery, apps, healthcare, retail services), not one narrow domain.
Aspect Taxonomy Used in the Challenge

food, service, price, cleanliness, delivery, ambiance, app_experience, general, none.

Use these classes consistently in your predictions to avoid schema or scoring mismatch.

Evaluation: Final scoring is done against a hidden labeled test set. Pillar 1 (30%) is automated using F1 (Micro)-based ABSA performance. Additional pillars score your data handling, technical approach, deployment/UI, and presentation quality. Your best valid submission should maximize both model quality and engineering clarity.


Your submission must be a <strong>.json</strong> file with the following structure for each review in the hidden test set:

[
